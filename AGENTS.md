# AGENTS.md

Guidance for any coding agent working in this repo. Read this before touching code.

## What this is

An interactive **D3.js map of the Eleken blog** (~530 articles) as an Obsidian-style dot field, clustered by publish year, with a sidebar tree, search, zoom/pan, hover labels, and click-to-focus. It is a **fully static page** — no backend, no client-side API calls at runtime. UI copy and the README are in **Ukrainian**.

Article data is pulled from **Webflow CMS at build time** on Vercel and baked into a static `nodes.js`. The committed `nodes.js` at the repo root is a snapshot so the map runs locally without a token.

## File map

| File | What it is | Who edits it |
|---|---|---|

| `public/` | Build output Vercel serves. Gitignored. | generated |

## Run / build / deploy

```bash
node scripts/serve.mjs            # http://localhost:8765 — map from the committed snapshot
WEBFLOW_TOKEN=xxx node scripts/build.mjs   # refresh data → public/
```

- Deploy is Vercel: `buildCommand: node scripts/build.mjs`, `outputDirectory: public` (see `vercel.json`). Build exits 1 without a token / on API error, so an empty map never deploys.
- A Webflow `site_publish` webhook → Vercel Deploy Hook rebuilds on publish. **Do not change this wiring.**

## Hard rules

- **Do not touch** the inlined D3 bundle (lines 1–3 of `eleken-blog-map.js`).
- Keep the **static runtime**: no backend, no client-side fetches, no framework, no build step beyond `scripts/`. Any "assistant / AI" feature must stay build-time-static (see the Gemini catalog plan below) — the RAG/vector-DB/serverless-proxy path was explicitly evaluated and rejected.
- `nodes.js` is **data, generated by the build** — never hand-edit its shape; logic lives only in `eleken-blog-map.js`.
- Data and logic stay in separate files so a data refresh never touches logic.
- There is **no test framework**. Verify in-browser (top-level `const`/`function` in the classic script are reachable from the devtools console) or with `node`-run assertion scripts. Reload, confirm a clean console, then commit.
- Commits are authored as `eleken-git` / `marketing@eleken.co`. If on `main`, branch first for non-trivial work.

### Active plans

- **Gemini-readable article catalog (sr-only)** — `docs/plans/2026-07-20-gemini-sr-only-catalog.md` (prompt: `docs/prompts/2026-07-20-gemini-sr-only-catalog.md`). Bakes a hidden, screen-reader-only catalog of all articles (title + Webflow body text + slug) into `index.html` during the existing build, so the browser's **built-in AI** (Chrome Ask Gemini / the Claude extension) can answer about the blog. Key facts: **no API key, no LLM, no GitHub Action** — the browser AI does all the AI at question time; the build just fetches text Webflow already has. The AI reads only the **rendered text snapshot** (so `sr-only`, never `display:none`); catalog is **newest-first** (truncation drops the oldest); body text stays out of `nodes.js`; refresh rides the existing Webflow→Vercel hook.
